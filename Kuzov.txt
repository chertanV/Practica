Модуль 1. Анализ требований, проектирование и разработка программных продуктов

Рабочее место №14

Постановка задания

В рамках выполнения практической работы необходимо разработать проект базы данных «Издательский центр», предназначенной для хранения, обработки и анализа информации о деятельности издательства.

Для выполнения задания требуется:

Разработать техническое задание на создание программного продукта «Издательский центр» в соответствии с требованиями ГОСТ 19.201-78 либо ГОСТ 34.602-89 (34.602-2020).
Выполнить проектирование базы данных «Издательский центр». Необходимо определить основные информационные объекты предметной области, описать их характеристики и атрибуты. При необходимости допускается добавление дополнительных атрибутов, которые позволят более полно представить структуру базы данных.
Представить информационные объекты в виде реляционных таблиц:

определить состав полей каждой таблицы;
указать тип данных каждого поля;
определить свойства полей;
выбрать первичный ключ для каждой таблицы;
определить внешние ключи, необходимые для связи таблиц между собой.

Определить типы связей между таблицами базы данных и построить логическую модель базы данных.

Исходная информация

База данных «Издательский центр»

Описание предметной области

Издательский центр «Печать» осуществляет сотрудничество с писателями на основании официальных контрактов, предусматривающих выпуск литературных произведений. Каждый автор в течение срока действия договора работает исключительно с данным издательством. При необходимости несколько авторов могут объединяться в творческие группы для совместной подготовки одной книги.

Издательский центр имеет право расторгнуть действующий контракт, продлить его либо заключить новый договор на иных условиях. После завершения работы над произведением издательство организует редакционную подготовку книги, ее печать, распространение и последующую реализацию заказчикам.

Заказчиками могут выступать различные организации, книжные магазины, библиотеки, образовательные учреждения, коммерческие компании и другие юридические лица. После реализации тиража авторы получают установленное договором вознаграждение (гонорар), размер которого зависит от условий контракта и количества проданных экземпляров.

Разрабатываемая база данных должна обеспечивать хранение всей необходимой информации, позволяющей автоматизировать работу издательского центра, повысить эффективность обработки данных и сократить время поиска необходимой информации.

⸻

Назначение базы данных

База данных предназначена для решения следующих задач:

хранение информации обо всех писателях;
учет заключенных контрактов;
регистрация выпущенных книг;
ведение учета заказчиков;
хранение информации о заказах;
расчет авторских гонораров;
контроль сроков действия контрактов;
учет затрат на выпуск книг;
анализ прибыли, полученной от продажи книг;
получение статистической информации по деятельности издательства.

Использование базы данных позволит автоматизировать большинство процессов, связанных с выпуском и реализацией книжной продукции, а также значительно уменьшить вероятность возникновения ошибок при обработке информации.

⸻

Основные информационные объекты

В базе данных должна храниться информация о следующих объектах:

Писатели

Для каждого писателя рекомендуется хранить следующие сведения:

уникальный идентификатор;
фамилия;
имя;
отчество;
дата рождения;
адрес проживания;
номер телефона;
адрес электронной почты;
дата заключения контракта;
статус сотрудничества.

⸻

Контракты

Информация о контрактах включает:

номер контракта;
дату заключения;
срок действия;
размер авторского гонорара;
условия сотрудничества;
статус контракта;
идентификатор писателя.

⸻

Книги

Для каждой книги должны храниться:

идентификатор книги;
название;
жанр;
дата издания;
тираж;
стоимость печати;
отпускная цена;
количество страниц;
идентификатор автора.

⸻

Заказчики

Для каждого заказчика необходимо хранить:

идентификатор;
название организации;
юридический адрес;
контактное лицо;
телефон;
электронную почту;
банковские реквизиты.

⸻

Заказы

Информация о заказах включает:

номер заказа;
дату оформления;
заказчика;
книгу;
количество экземпляров;
стоимость заказа;
дату поставки;
статус выполнения заказа.

⸻

Ожидаемый результат

После завершения проектирования должна быть получена полноценная структура базы данных, позволяющая эффективно хранить и обрабатывать информацию о деятельности издательского центра.
