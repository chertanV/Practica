## Страницы и функционал

### 1. Гость (до входа)
- **Вход** (`/login`) — email + пароль
- **Регистрация** (`/register`) — без подтверждения email. После регистрации в Firestore создаётся профиль в коллекции `users` (документ с id = uid), роль по умолчанию — **пустая** (доступ появится после назначения администратором).

### Роли (admin / seller)
- **Админ** — полный доступ: главная, отчёты, продажи, товары, поставщики.
- **Продавец** — главная, продажи, товары (без разделов «Отчёты», «Поставщики»). Импорт из Excel доступен на странице «Товары». При прямом переходе по URL на страницу только для админа происходит перенаправление на главную.
- Роль хранится в Firestore в документе `users/{uid}` (поле `role`: `"admin"` или `"seller"`).
- **Как назначить первого админа:** в консоли Firebase (Firestore) открыть коллекцию `users`, найти документ с id = uid пользователя (uid можно взять в разделе Authentication), и в поле `role` задать значение `admin`. Либо создать документ с id = uid и полями `email`, `role: "admin"`.
- Если у пользователя в `users/{uid}` роль ещё не назначена (поле `role` пустое) — приложение показывает страницу ожидания `waiting`, а доступ к остальным разделам запрещён до назначения роли.

### 2. Главная (`/main`)
- **Сводка:** общая выручка, количество заказов (чеков), активные товары
- **Нет в наличии** — список товаров с нулевым остатком
- **Быстрые действия** — кнопки перехода к разделам

### 3. Товары (`/products`)
- Таблица: ID, название, поставщик, описание, штрих-код, категория, цены закупки/продажи, остаток
- Поиск по названию, описанию, штрих-коду
- Фильтры по поставщику и категории
- Добавление, редактирование, удаление товаров
- **Импорт из Excel** — выбор файла и кнопка «Загрузить» на этой же странице. Товары ищутся по штрих-коду, затем по названию:
- При импорте, если поставщик из строки Excel отсутствует в коллекции `suppliers`, он создаётся автоматически. Остальные поля (`contactPerson`, `contractNumber`, `address`) остаются пустыми.
- Товар считается «тем же», только если совпадают:
  - `поставщик`,
  - `цена закупки`,
  - `цена продажи`,
  - и идентификатор: `штрих-код` (если он заполнен) иначе `название`.
  - при совпадении — обновляются непустые поля и **остаток увеличивается** (а не создаётся новый документ);
  - если поставщик/штрих-код/цены отличаются — создаётся **другой** товар.
- Если в строке Excel недостаточно данных для однозначного определения товара (нет поставщика, цен или штрих-кода/названия) — строка пропускается с сообщением.
- Если в строке Excel не заполнены некоторые поля (например, описание/категория), то при обновлении эти поля **не перезаписываются пустыми** — сохраняются значения из БД.
- При ручном создании товара применяется тот же принцип: слияние выполняется только при совпадении `поставщик` + обе цены + идентификатор (`штрих-код` если задан, иначе `название`).
- При импорте и ручном создании применяется нормализация регистра для человекочитаемых полей:
  - `ооо`/`ип` приводятся к `ООО`/`ИП`
  - остальной текст приводится к виду `Тест/Компания` (первая буква заглавная, остальное строчное)
- Кнопка «Продать» — оформление продажи (создание продажи по одному товару из карточки).

### 4. Продажи (`/sales`)
- Продажи ведутся **по чекам**:
  - один чек = один документ в `sales` с полем `items` (массив позиций: товар, количество, цены, скидка);
  - старые продажи без `items` автоматически приводятся к одному товару в чеке.
- Журнал: № чека, дата, продавец, способ оплаты, количество позиций, сумма чека.
- Фильтры по периоду и поиску (по товару и продавцу).
- Создание нового чека: список позиций (товары и количества) + общие поля (дата, продавец, способ оплаты), списание остатков.
- Просмотр чека: по кнопке «Открыть» видно состав чека (товары, количество, цены, скидки, итог).
- Удаление чека: по кнопке «Удалить» чек удаляется, а все товары из него **возвращаются на склад** (увеличиваются остатки).

### 5. Поставщики (`/suppliers`)
- Таблица: ID, ФИО/название, контактное лицо, номер договора, адрес
- Добавление, редактирование, удаление
- При создании/редактировании поставщика название нормализуется (без учёта регистра и пробелов по краям), дубликаты по названию запрещены.

### 6. Отчёты (`/reports`)
- **Детальный отчёт:** период, название, поставщик, продажи, цена закупки, цена продажи, выручка
- **По контрагентам:** сводка по поставщикам
- Фильтры: даты, поставщик, мин. продаж (не меньше 1)
- Отчёт пересчитывается автоматически при изменении фильтров; кнопка «Сгенерировать» не используется.
- Вывод в таблице и экспорт в Excel (скидка учитывается за счёт поля `discountedPrice`).

## Firebase Firestore — коллекции

| Коллекция | Назначение |
|-----------|------------|
| `products` | Товары (название, поставщик, описание, штрих-код, цены, остаток) |
| `sales` | Все продажи: чеки с полем items[] (позиции) или старый формат — один документ на позицию (товар, кол-во, цены, скидка, способ оплаты, продавец, дата). Отдельной коллекции salesLog нет. |
| `suppliers` | Поставщики (ФИО, контакт, договор, адрес) |
| `users` | Пользователи (если используется) |

## Firebase Auth

- Регистрация без подтверждения email
- Вход по email и паролю
- Сообщения об ошибках на русском (неверный пароль, email занят и т.д.)

## Запуск проекта

1. Запустить локальный сервер (из корня проекта):
   ```bash
   npx serve .
   ```
   или через Live Server (порт 5501).

2. Открыть в браузере: `http://localhost:3000` (или указанный порт).

3. В Firebase Console:
   - Включить **Authentication → Email/Password**
   - Создать **Firestore Database**
   - Настроить правила доступа

## Основные правила

- **Цена продажи** должна быть выше цены закупки, иначе работа в ноль.
- При импорте — если цена продажи ≤ закупки, автоматически выставляется +10%.
- **Штрих-код** — строка (EAN-13, EAN-8 и др.).
- **Тема** (светлая/тёмная) сохраняется в `localStorage`.
