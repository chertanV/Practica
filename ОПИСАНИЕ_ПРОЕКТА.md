# Система учёта продаж — Описание проекта

Документ описывает структуру проекта, реализованный функционал и назначение файлов.

## Общая информация

**Название:** Система учёта продаж  
**Технологии:** HTML, CSS, JavaScript (ES Modules), Firebase (Auth + Firestore), XLSX  
**Архитектура:** MVVM (Model — View — ViewModel)  
**Тип:** SPA (Single Page Application) — одностраничное приложение

## Страницы и функционал

### 1. Гость (до входа)
- **Вход** (`/login`) — email + пароль
- **Регистрация** (`/register`) — без подтверждения email

### 2. Главная (`/main`)
- **Сводка:** общая выручка, количество заказов, активные товары
- **Нет в наличии** — список товаров с нулевым остатком
- **Быстрые действия** — кнопки перехода к разделам

### 3. Товары (`/products`)
- Таблица: ID, название, поставщик, описание, штрих-код, категория, цены закупки/продажи, остаток
- Поиск по названию, описанию, штрих-коду
- Фильтры по поставщику и категории
- Добавление, редактирование, удаление товаров
- Кнопка «Продать» — оформление продажи

### 4. Продажи (`/sales`)
- Журнал: №, наименование, кол-во, полная цена, скидка, цена со скидкой, способ оплаты, ФИО продавца, дата и время
- Фильтры по периоду и поиску
- Новые продажи

### 5. Поставщики (`/suppliers`)
- Таблица: ID, ФИО, контактное лицо, номер договора, адрес
- Добавление и редактирование

### 6. Отчёты (`/reports`)
- **Детальный отчёт:** период, название, поставщик, продажи, цена закупки, цена продажи, выручка
- **По контрагентам:** сводка по поставщикам
- Фильтры: даты, поставщик, мин. продаж
- Вывод в таблице и экспорт в Excel

### 7. Импорт (`/imports`)
- Загрузка Excel: товары или продажи
- Товары: название, поставщик, описание, штрих-код, категория, цены, остаток
- Продажи: название, поставщик, количество, дата, цена

## Firebase Firestore — коллекции

| Коллекция | Назначение |
|-----------|------------|
| `products` | Товары (название, поставщик, описание, штрих-код, цены, остаток) |
| `sales` | Продажи (товар, кол-во, цены, скидка, способ оплаты, продавец, дата) |
| `salesLog` | Лог продаж (продавец, дата, товар, кол-во, №) |
| `suppliers` | Поставщики (ФИО, контакт, договор, адрес) |
| `users` | Пользователи (если используется) |

## Firebase Auth

- Регистрация без подтверждения email
- Вход по email и паролю
- Сообщения об ошибках на русском (неверный пароль, email занят и т.д.)

## Запуск проекта

1. Запустить локальный сервер (из корня проекта):
   ```bash
   npx serve .
   ```
   или через Live Server (порт 5501).

2. Открыть в браузере: `http://localhost:3000` (или указанный порт).

3. В Firebase Console:
   - Включить **Authentication → Email/Password**
   - Создать **Firestore Database**
   - Настроить правила доступа

---

## Основные правила

- **Цена продажи** должна быть выше цены закупки, иначе работа в ноль.
- При импорте — если цена продажи ≤ закупки, автоматически выставляется +10%.
- **Штрих-код** — строка (EAN-13, EAN-8 и др.).
- **Тема** (светлая/тёмная) сохраняется в `localStorage`.
